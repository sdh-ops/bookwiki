# 베스트셀러 시스템 개선 사항

## 📋 개선 개요

기존 6개 카테고리에서 **10개 카테고리**로 확대하고, 모든 스크레이퍼의 문제점을 수정했습니다.

---

## 🎯 주요 개선 사항

### 1. **카테고리 대폭 확대** (6개 → 10개)

#### 기존 카테고리
- 종합
- 소설
- 에세이/시
- 인문
- 경제경영
- 자기계발

#### 추가된 카테고리
- 역사
- 사회과학
- 과학
- 어린이/청소년

### 2. **알라딘 API 적용**

기존의 웹 스크래핑 방식에서 **공식 API**로 전환:
- ✅ 더 안정적인 데이터 수집
- ✅ 속도 향상
- ✅ 서버 부하 감소
- ✅ ISBN 정보 자동 수집

**API 엔드포인트**: `ItemList.aspx` (Bestseller 타입)

### 3. **밀리의서재 스크레이퍼 완전 수정**

#### 기존 문제
- Strategy 3에서 이미지 alt 속성으로 책 제목을 가져오다가 UI 요소를 잘못 수집
- "밀리의서재 로고", "rank1 crown", "서점 베스트셀러" 등 UI 텍스트가 책 제목으로 저장됨

#### 개선 사항
- ✅ Strategy 3 완전 재작성
- ✅ 책 링크 기반 수집 (가장 안정적)
- ✅ 다중 셀렉터 전략으로 안정성 향상
- ✅ UI 요소 필터링 (로고, 아이콘 등 제외)
- ✅ 중복 제목 자동 제거

### 4. **리디북스 카테고리 수정**

#### 기존 문제
- "종합" 카테고리에서 `general` 코드 사용 시 0개 데이터 수집

#### 개선 사항
- ✅ 종합 카테고리는 메인 베스트셀러 페이지(`/bestsellers/`)로 접근
- ✅ 데이터 파싱 로직 개선 (여러 쿼리 경로 시도)

### 5. **YES24 셀렉터 개선**

#### 기존 문제
- 8개만 수집되는 문제

#### 개선 사항
- ✅ 다중 셀렉터 전략 적용
  ```javascript
  const itemSelectors = [
    '#yesBestList > li',
    '.itemUnit',
    '.goods_info',
    '#bestList li'
  ];
  ```
- ✅ 20개 전체 수집 보장

### 6. **중복 방지 개선**

- ✅ 제목 정리 함수 추가 (`[도서]` 태그 제거)
- ✅ 저자명 정리 함수 개선
- ✅ 스냅샷 저장 시 중복 자동 처리
- ✅ 정규화 스크립트로 사후 처리

## 📂 새로 생성된 파일

### 1. `scripts/wipe-data.js`
모든 베스트셀러 데이터 삭제 (스냅샷 + 책 정보)

### 2. `scripts/bestseller-v2.js`
완전히 개선된 스크레이퍼:
- 알라딘 API 사용
- 10개 카테고리 지원
- 모든 버그 수정

### 3. `scripts/rebuild-all.js`
전체 재구축 자동화 스크립트:
1. 데이터 삭제
2. 새 스크레이퍼 실행
3. 제목 정규화
4. 데이터 검증

---

## 🚀 실행 방법

### 전체 재구축 (권장)
```bash
node scripts/rebuild-all.js
```

### 단계별 실행
```bash
# 1. 데이터 삭제
node scripts/wipe-data.js

# 2. 새 데이터 수집
node scripts/bestseller-v2.js

# 3. 정규화
node scripts/normalize-books.js

# 4. 검증
node scripts/check-platform-data.js
```
